let & const instead of var

  • var hoists, let and const don't
  • Readability: you always see where you have mutable or immutable variables

let & const

			if (condition) {
			    let name = 'Ann';
			    const users = [];
			    ...
			}
			alert(name); // name is not defined
			alert(users); // users is not defined

One var/let/const per line

			// bad
			var type = 'Binary',
			    data = [ ... ];
		
			// good
			var type = 'Binary';
			var data = [ ... ];

One var/let/const per line

			// good
			const RADIUS = 13;
			const someInterestingNumber = 3.14;
			let pointer = 0;
